PMS Adapter Layer

Abstraction interface รองรับหลาย PMS — เพิ่ม adapter ใหม่ได้โดยไม่แก้ core logic

Interface Contract (Abstract Base Class)

class
PMSAdapter
(ABC):
# ทุก adapter ต้อง implement 3 methods นี้
async def verify_guest(room: str, last_name: str) → GuestInfo | None
→ ตรวจสอบว่า guest check-in อยู่จริง, คืน GuestInfo ถ้าถูก
async def get_guest_by_room(room: str) → GuestInfo | None
→ ดึงข้อมูล guest ปัจจุบันใน room
async def health_check() → bool
→ ตรวจสอบ connectivity กับ PMS
# GuestInfo dataclass
@dataclass
class GuestInfo:
pms_id: str
room_number: str
last_name: str
check_in: datetime
check_out: datetime

Adapter Factory & Fallback

Factory Pattern
โหลด adapter จาก pms_adapters table ที่ is_active=true → inject เข้า auth service ตอน startup
Fallback Strategy
PMS timeout/error → retry 2 ครั้ง → log error → คืน auth failure พร้อม error message ชัดเจน (ไม่ fallback เงียบ)
