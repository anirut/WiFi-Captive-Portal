Authentication Flow & Network Control

Guest journey ตั้งแต่เชื่อม WiFi จนใช้อินเทอร์เน็ตได้

Flow A — Room + Last Name

1
Guest เชื่อม WiFi → dnsmasq ให้ IP
2
เปิด browser → iptables redirect port 80/443 → FastAPI portal
3
กรอก Room Number + Last Name + ยอมรับ T&C
4
FastAPI → PMS Adapter ตรวจสอบ guest กำลัง check-in จริง
5
ดึง policy จาก room type → สร้าง session → บันทึก IP/MAC
6
iptables ADD IP → ACCEPT chain + bandwidth limit (tc)
7
Redirect ไปหน้า success → ใช้อินเทอร์เน็ตได้

Flow B — Voucher Code

1
Guest เชื่อม WiFi → iptables redirect → portal
2
เลือก "Voucher Code" → กรอกรหัส
3
ตรวจสอบ: ถูกต้อง, ไม่หมดอายุ, ยังเหลือ uses
4
สร้าง session → ดึง policy จาก voucher type
5
iptables ADD IP → ACCEPT + bandwidth limit
6
ใช้อินเทอร์เน็ตตาม type: หมดเวลา หรือ หมด data → expire

Session Expiry (Background Task)

Check-out Sync
PMS แจ้ง check-out → FastAPI expire session ของ room นั้นทันที → iptables REMOVE
Timer Expiry
background scheduler ตรวจ expires_at ทุก 60 วิ → REMOVE ที่หมดอายุ
Manual Kick
Admin กด kick จาก dashboard → iptables REMOVE ทันที → session status = kicked
iptables Rules Overview
# Redirect unauthenticated traffic to portal
iptables -t nat -A PREROUTING -i wlan0 -p tcp --dport 80 -j DNAT --to 192.168.1.1:8080
# Allow authenticated IP (added per session)
iptables -I FORWARD -s <guest_ip> -j ACCEPT
# Remove on expire/kick
iptables -D FORWARD -s <guest_ip> -j ACCEPT
