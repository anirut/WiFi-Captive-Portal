## Global Enterprise PMS

These are major players used by hotel chains and large properties worldwide.

### 1. Oracle Opera PMS (MICROS)
- **Website:** oracle.com/hospitality
- **Type:** Enterprise, on-premises/cloud hybrid
- **Market Share:** Dominates luxury and enterprise segment
- **API:** Opera WebAPI, REST APIs, FIAS
- **Integration Complexity:** High (legacy protocols, complex setup)
- **Typical Clients:** Luxury hotels, hotel chains, resorts
- **Notes:** One of the most widely deployed enterprise PMS globally

### 2. Amadeus SynXis
- **Website:** amadeus.com
- **Type:** Cloud-based enterprise
- **Market Share:** Major global player
- **API:** REST APIs, OTA standards
- **Integration Complexity:** Medium-High
- **Typical Clients:** Hotel chains, management companies
- **Notes:** Part of Amadeus travel tech ecosystem; strong CRS integration

### 3. Infor HMS (Hospitality Management System)
- **Website:** infor.com/hospitality
- **Type:** Enterprise, cloud and on-prem
- **Market Share:** Significant in upscale segment
- **API:** Infor API framework, REST
- **Integration Complexity:** Medium-High
- **Typical Clients:** Chain hotels, select service
- **Notes:** Strong multi-property support

### 4. SAP Hospitality
- **Website:** sap.com/hospitality
- **Type:** Enterprise ERP-integrated
- **Market Share:** Strong in Europe
- **API:** SAP APIs, RFC/BAPI
- **Integration Complexity:** High
- **Typical Clients:** Large chains, integrated ERP users
- **Notes:** Part of SAP ecosystem; very large deployments

### 5. Shiji Enterprise Solutions
- **Website:** shiji.com
- **Type:** Enterprise, cloud and on-prem
- **Market Share:** Growing globally, strong in Asia
- **API:** Shiji SPAS (JSON REST API)
- **Integration Complexity:** Medium
- **Typical Clients:** Large chains, casino hotels
- **Notes:** Acquired many PMS brands (MICROS, Infor HMS, etc.); strong APAC presence

### 6. TravelClick (now Amadeus)
- **Website:** amadeus.com/hospitality
- **Type:** Cloud, mid-to-enterprise
- **Market Share:** North America strong
- **API:** REST APIs
- **Integration Complexity:** Medium
- **Typical Clients:** Mid-large chains
- **Notes:** Now fully integrated into Amadeus

## Thailand PMS Market

ประเทศไทยเป็นหนึ่งในจุดหมายปลายทางการท่องเที่ยวที่สำคัญที่สุดในโลก (อันดับ 1 เอเชียตะวันออกเฉียงใต้, ต้อนรับนักท่องเที่ยว ~35-40 ล้านคน/ปี) มีโรงแรมและรีสอร์ทจำนวนมากที่ต้องการระบบ PMS

### ผู้ให้บริการ PMS สากลที่นิยมในไทย

| PMS | ประเภท | สัดส่วนโดยประมาณ | API |
|-----|--------|------------------|-----|
| Oracle Opera / MICROS | Enterprise | ~25-30% | FIAS/REST |
| Mews | Cloud-native | ~15-20% | REST + Webhooks |
| Cloudbeds | Cloud | ~10-15% | REST + Webhooks |
| Shiji (รวม MICROS) | Enterprise | ~10-15% | SPAS API |
| Amadeus SynXis | Cloud | ~5-10% | REST |
| Little Hotelier | Cloud | ~5-10% | Webhooks |

### หมายเหตุเกี่ยวกับ Local PMS ไทย

**ไม่สามารถยืนยันข้อมูลผู้ให้บริการ PMS ท้องถิ่นของไทย (Local PMS) ได้อย่างถูกต้อง**

ข้อมูลที่พบในการค้นคว้ามีดังนี้:
- ผู้ให้บริการ PMS ในไทยส่วนใหญ่เป็นตัวแทน (Reseller) ของ PMS สากล มากกว่าจะเป็นผู้พัฒนาเอง
- บางระบบอาจเป็น Custom-built สำหรับโรงแรมเฉพาะ ไม่ได้เป็น Commercial PMS
- ข้อมูลเกี่ยวกับ Local PMS ที่มี API สำหรับ Integration ยังไม่ชัดเจน

### แนะนำสำหรับ Integration

1. **ใช้ Mews หรือ Cloudbeds** - ทั้งสองมี API ที่ดีและมีการใช้ในไทย
2. **ติดต่อตัวแทน PMS ในไทย** - บริษัทอย่าง Kratos, MTC, หรือตัวแทนอื่นๆ อาจช่วยหา Solution ที่เหมาะสม
3. **Custom Integration** - สำหรับ Local PMS ที่ไม่มี API อาจต้องพัฒนา Custom Adapter
